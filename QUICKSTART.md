# 快速启动指南

## 一键启动

```bash
./start.sh
```

## 访问地址

- 🎨 前端: http://localhost:5173
- 🔧 后端: http://localhost:3001
- ❤️  健康检查: http://localhost:3001/health

## 默认数据

系统预置了3个示例项目：

1. **Moma** - 管理后台系统
   - 包含2个Git仓库（前端+后端）

2. **Codelink** - 代码协作平台
   - 包含1个核心服务仓库

3. **AI需求交付平台** - 本项目
   - 包含1个主仓库

## 5分钟体验完整流程

### 1️⃣ 查看项目资产 (1分钟)
- 进入 **"我的资产"** 页面
- 查看预置的3个项目
- 点击项目查看Git仓库信息

### 2️⃣ 创建需求 (1分钟)
- 进入 **"首页"**
- 选择项目：Moma
- 输入需求：
  ```
  在用户列表页添加搜索功能，支持按用户名和邮箱搜索
  ```
- 点击"提交需求"

### 3️⃣ 会话交互 (2分钟)
- 自动跳转到任务详情页
- 在对话框输入补充信息
- 查看AI回复（模拟）
- 观察任务状态变化

### 4️⃣ 查看报告 (1分钟)
- 点击"完成任务"按钮
- 右侧查看交付报告
- 查看代码变更、数据库变更、配置变更
- 点击MR链接

## 常见问题

### Q: 端口被占用怎么办？
A: 修改端口配置：
```bash
# 修改后端端口
server/.env: PORT=3002

# 前端会自动分配可用端口
```

### Q: 如何停止服务？
A: 在终端按 `Ctrl+C`

### Q: 数据会保存吗？
A: 当前使用Mock数据，重启后数据会重置。要持久化数据需要连接真实数据库。

### Q: 如何连接真实数据库？
A:
1. 安装PostgreSQL
2. 修改 `server/.env` 中的 `DATABASE_URL`
3. 运行 `npm run prisma:migrate`
4. 修改控制器代码，使用Prisma Client替代Mock数据

## 停止服务

```bash
# 如果使用start.sh，直接Ctrl+C
# 或手动停止进程
pkill -f "ts-node-dev"
pkill -f "vite"
```
